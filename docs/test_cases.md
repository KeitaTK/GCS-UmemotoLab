# 統合・検証フェーズ テストケース設計

## 1. テレメトリー検証（1台ドローン）
- ドローン1台を接続し、テレメトリー情報が正しく受信・記録されるか確認
- ログにハートビート、コマンド送信成功が記録されるか

## 2. システムIDごとのルーティング検証（2台ドローン）
- ドローン2台を接続し、各システムIDごとにルーティングが正しく行われるか
- 各ドローンのテレメトリーが混在せず、正しいIDで記録されるか

## 3. RTCMインジェクション動作確認
- u-center経由でRTCMデータをインジェクションし、ドローン側で受信できるか
- ログにRTCMインジェクション成功が記録されるか

## 4. 設定ファイル検証
- `config/gcs.yml`の設定変更がシステムに反映されるか
- 設定値ごとに動作が変化するか

## 5. 複数ドローン動作確認
- 2台以上のドローンを同時接続し、全ての機能が正常動作するか
