# 🚁 GCSプロジェクト 飛行ロードマップ

> **ドローン1台を実際に飛行させるまでの全工程**
>
> **作成日**: 2026-07-01
> **最終更新**: 2026-07-01
> **対象機体**: Pixhawk 6C + Raspberry Pi 5 + u-blox ZED-F9P
> **GCS環境**: MacBook Air (launchd常時稼働) + Web UI (localhost:8000)

---

## 1. ロードマップ全体像

### 1.1 現在地とゴール

```
【現在地】                              【ゴール】
RTK Fixed (fix=6) 達成済み          ──→  1台のドローンが自律飛行し、
GCS遠隔操作・テレメトリ完備              cm級精度の位置保持を実証する
```

### 1.2 フェーズ構成

| フェーズ | 名称 | 目標 | 期間目安 | 依存関係 |
|---------|------|------|----------|----------|
| **A** | 飛行前チェック | 機体・FCパラメータ・センサーの完全確認 | 1〜2日 | なし（現状から即開始可） |
| **B** | 地上テスト | プロペラなしでの全動作確認（GCS遠隔操作） | 1日 | A 完了後 |
| **C** | 初飛行 | 低高度ホバリング・手動/自動切替確認 | 1〜2日 | B 完了後 |
| **D** | 自律飛行テスト | NED精密移動・ウェイポイント・RTL実証 | 2〜3日 | C 完了後 |
| **E** | 安全確認・チューニング | PID最適化・振動解析・緊急停止検証 | 2〜4日 | D 完了後 |

### 1.3 全体タイムライン

```
Week 1          Week 2          Week 3          Week 4
| A: 飛行前     | B: 地上        | C: 初飛行      | D: 自律飛行    |
| チェック      | テスト         |                |                |
|               |                |                | E: 安全確認    |
|               |                |                | チューニング   |
└─ 屋内作業 ────┴─ 屋内作業 ────┴─ 屋外飛行 ────┴─ 屋外飛行 ────┘
```

### フェーズA: 飛行前チェック 🛠️

> **目標**: 機体のハードウェア・ソフトウェアが飛行可能な状態であることを確認する。
> **場所**: 屋内（机上学習・設定確認）、屋外（GPS/コンパス校正時のみ）
> **期間目安**: 1〜2日

#### A-1. 機体ハードウェア確認

| # | チェック項目 | 方法 | 合否基準 |
|---|-------------|------|----------|
| A1.1 | モーター回転方向・固定 | 手で回して異音なし、全モーター同方向（CW/CCW交互）確認 | 異音なし・スムーズ回転 |
| A1.2 | ESC接続・設定 | 各ESCがPixhawk PWM出力に接続、キャリブレーション済み | 4chとも信号線接続確認 |
| A1.3 | プロペラ状態 | キズ・ひび割れ・バランス確認 | 損傷なし・バランス良好 |
| A1.4 | バッテリー電圧・セルバランス | テスターで実測 / GCSテレメトリで確認 | セル間差 0.1V以内 |
| A1.5 | フレーム剛性・ネジ緩み | 全ネジの増し締め、アームのガタつき確認 | 緩みなし |
| A1.6 | 配線の固定・干渉 | プロペラ可動域に配線が入らないか目視確認 | 干渉なし・タイラップ固定済み |
| A1.7 | 機体重量・重心 | バッテリー位置調整で重心が機体中心にあるか | 水平保持可能 |
| A1.8 | Pixhawk 6C マウント | 防振マウントの状態、ケースの固定 | ガタつきなし・ダンパー劣化なし |

#### A-2. フライトコントローラ（Pixhawk 6C）パラメータ確認

> **参照**: `GPS_TYPE=9` (DroneCAN), `GPS_AUTO_CONFIG=2`, `GPS_INJECT_TO=127` は設定済み

| # | パラメータ | 推奨値 | 確認方法 | 備考 |
|---|-----------|--------|----------|------|
| A2.1 | `AHRS_EKF_TYPE` | `2` (EKF2) または `3` (EKF3) | QGC / GCS params API | EKF3 healthy確認済み |
| A2.2 | `ARMING_CHECK` | `1`（全チェック有効） | QGCパラメータ | ⚠️ 実飛行では必ず有効に戻す |
| A2.3 | `FS_THR_ENABLE` | `1`（ラジオフェールセーフ有効） | QGCパラメータ | 緊急時自動RTLに必須 |
| A2.4 | `FS_BATT_ENABLE` | `1`（バッテリーフェールセーフ有効） | QGCパラメータ | 要: `FS_BATT_VOLTAGE`, `FS_BATT_MAH` 設定 |
| A2.5 | `FS_BATT_VOLTAGE` | 3.5V/セル × セル数 | QGCパラメータ | e.g. 4S→14.0V |
| A2.6 | `FS_BATT_MAH` | バッテリー容量の80% | QGCパラメータ | e.g. 5000mAh→4000mAh |
| A2.7 | `FS_GCS_ENABLE` | `1`（GCS通信断検知） | QGCパラメータ | GCS Heartbeat途絶時の動作 |
| A2.8 | `FENCE_ENABLE` | `1`（ジオフェンス有効） | QGCパラメータ | 要: フェンス半径・上限高度設定 |
| A2.9 | `FENCE_ALT_MAX` | テスト時の上限高度（10〜20m推奨） | QGCパラメータ | 初飛行は10m以下推奨 |
| A2.10 | `FENCE_RADIUS` | テストエリア半径（30〜50m推奨） | QGCパラメータ | 飛行場所の広さに合わせる |
| A2.11 | `RTL_ALT` | 周囲の障害物より高い高度 | QGCパラメータ | 最低10m推奨 |
| A2.12 | `MOT_SPIN_ARM` | `0.1`（アーム時の低速回転） | QGCパラメータ | アーム確認用 |
| A2.13 | `MOT_SPIN_MIN` | デフォルト値確認 | QGCパラメータ | 最低回転数 |
| A2.14 | `PILOT_SPEED` | 手動飛行の最大速度（初回は5m/s以下推奨） | QGCパラメータ | |
| A2.15 | `PILOT_SPEED_UP` | 手動上昇速度（初回は2m/s以下推奨） | QGCパラメータ | |

#### A-3. フェールセーフ設定 詳細

```
【フェールセーフ 全体フロー】
                              ┌─ RTL実行 ───→ 離陸地点へ帰還
通信断 ──→ FS_GCS_ENABLE ──→ │
                              ├─ Land実行 ──→ その場で着陸
                              │
                              ┌─ RTL実行 ───→ 離陸地点へ帰還
バッテリー低下 ──→ FS_BATT_ENABLE ─→│
                              ├─ Land実行 ──→ その場で着陸
                              │
ジオフェンス逸脱 ──→ FENCE_ENABLE ──→ RTL / Land
```

| # | フェールセーフ項目 | 設定値 | 動作 | 確認方法 |
|---|-------------------|--------|------|----------|
| A3.1 | **GCS通信断** | `FS_GCS_ENABLE=1` | 5秒間GCS heartbeat途絶→RTL | 地上テストでGCSサーバー停止して確認 |
| A3.2 | **バッテリー低下(電圧)** | `FS_BATT_VOLTAGE=14.0` (4S) | 設定電圧以下→Land/RTL | バッテリー放電テスト |
| A3.3 | **バッテリー低下(容量)** | `FS_BATT_MAH=4000` (例) | 設定容量消費→Land/RTL | 飛行ログで確認 |
| A3.4 | **ジオフェンス** | `FENCE_RADIUS=50`, `FENCE_ALT_MAX=20` | 逸脱→RTL/着陸 | QGC地図上でフェンス表示確認 |
| A3.5 | **ラジコン電波断** | `FS_THR_ENABLE=1` | スロットル信号途絶→RTL | ラジコン電源OFFで確認（地上テスト） |
| A3.6 | **EKF異常** | `EK3_FAILSAFE=1` | EKF発散→Land | パラメータ確認 |
| A3.7 | **RTL着陸後自動ディスアーム** | `DISARM_DELAY=5` | 着陸5秒後に自動ディスアーム | パラメータ確認 |

#### A-4. GPS ロック・RTK Fixed 安定確認

> **前提**: サンプル5にてRTK Fixed (fix=6) 達成済み。20サンプルで座標完全安定を確認。

| # | チェック項目 | 方法 | 合否基準 |
|---|-------------|------|----------|
| A4.1 | GPS 3D Fix 取得時間 | 電源投入→GCSでfix type確認 | 3分以内に fix≥3 |
| A4.2 | RTCM注入開始〜fix=4到達 | GCSログでGPS_RAW_INT監視 | RTCM注入後30秒以内にfix=4 |
| A4.3 | RTK Fixed (fix=6) 到達 | GCSログ / Web UI のRTK status | 注入後2〜5分以内にfix=6 |
| A4.4 | RTK Fixed 持続安定性 | 5分間 fix=6 維持を確認 | fix=6 が途切れない |
| A4.5 | 衛星数 | GCSテレメトリで `satellites_visible` | 15以上（RTKには最低5共通衛星必要） |
| A4.6 | HDOP | GCSテレメトリで `eph` / `epv` | HDOP < 1.0 |
| A4.7 | u-blox基準局 NMEA出力有無 | `rtk_base_station.py` の出力確認 | NMEA出力あり（Survey-In mode推奨）⚠️ |
| A4.8 | RTCM注入安定性 | GCSログでRTCM注入ログ連続確認 | 切断なし（localhost化済みのため安定が期待） |

> ⚠️ **A4.7重要**: 現状u-bloxがTIME mode（NMEA未出力）。絶対誤差測定にはSurvey-In mode切替が必要。飛行テストには必須ではないが、位置精度の客観的評価のために推奨。
