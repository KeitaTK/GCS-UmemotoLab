## 1. 概要（プロジェクトの目的）

### 1.1 このプロジェクトで作っているもの

**ドローンを遠隔で操作・監視するための地上管制システム（Ground Control Station = GCS）**です。

簡単に言うと、**パソコンやブラウザからドローンの状態を見たり、飛行指示を出したりできるシステム**です。

### 1.2 なぜ必要なのか

| 課題 | 解決策 |
|---|---|
| 複数のドローンを同時に管理したい | 1つの画面で全機の状態を一覧表示 |
| GPS精度をcm級にしたい | RTK補正技術で高精度測位を実現 |

### 1.3 システム全体像

```
┌─────────────┐     Tailscale(VPN)     ┌──────────────┐     UART     ┌──────────┐
│  PC/Mac     │ ◄──────────────────► │  Raspberry Pi │ ◄─────────► │ Pixhawk  │
│  (GCS画面)  │    インターネット経由    │  (通信中継)    │   シリアル   │ (ドローン)│
└─────────────┘                       └──────────────┘             └──────────┘

補足:
- Tailscale = 安全なVPN。離れた場所の機器を同じネットワーク内のように接続できる技術
- Raspberry Pi = 小型コンピュータ。ドローンの近くに設置し、通信を中継する役割
- Pixhawk = ドローンの頭脳。GPSやセンサーの情報を集め、飛行を制御する
- UART = シリアル通信規格。Raspberry PiとPixhawkを有線で接続
```

### 1.4 主な機能

| 機能 | 説明 |
|---|---|
| 📡 **テレメトリ受信** | ドローンのGPS位置・高度・速度・バッテリー残量などをリアルタイム表示 |
| 🎮 **機体制御** | アーム（プロペラ始動）・離陸・着陸・移動指示を遠隔から実行 |
| 🛰️ **RTK高精度測位** | GPSの誤差をcm級に補正する技術。専用アンテナ（u-blox）を使用 |
| 📊 **マルチドローン対応** | 複数機を1つの画面で同時に管理 |
| 🌐 **ブラウザ対応** | Webブラウザだけで操作可能。ソフトのインストール不要 |

### 3.2 通信構成

```
屋外                       屋内                    
┌──────────┐              ┌──────────┐
│ Raspi +  │  Tailscale   │ Mac mini │
│ Pixhawk  │◄────VPN────►│ (サーバー)│
│ u-blox   │              │          │
└──────────┘              └────┬─────┘
                               │
                    ブラウザからアクセス
                    http://100.75.83.95:8000/
```

**Tailscale** というVPNを使うことで、屋外の機器と屋内のサーバーを**安全に直接接続**できます。大学のネットワーク制限があっても通信可能です。

### 4.1 Web操作画面

![Web UI]()

ブラウザで動作するドローン管理画面を開発しました。

**実装済み機能:**

- RTKステータス表示
- NED座標（ローカル座標系）表示

- 未接続時のグレーアウト・操作ロック
- Connectボタン点滅（接続促進）

**安全設計:**
- LAND ALLボタンを赤→青に変更（赤は緊急停止専用に）
- ARM ALL/DISARM ALL/TAKEOFF ALL に確認ダイアログ
- 離陸高度に「Alt (m)」ラベル表示
- 未接続時は全操作ボタンを無効化

### 4.2 RTK補正システム

RTK（Real-Time Kinematic）は、GPSの誤差を**cm級**まで改善する技術です。

**仕組み:**
1. 位置が正確にわかっている基準局（u-bloxアンテナ）が「GPS衛星の信号は今これだけずれている」という補正データを生成
2. その補正データをドローンのGPSに送信
3. ドローン側が補正を適用 → 誤差が1m→1cmに改善

**構築済みの流れ:**
```
u-blox → シリアル通信 → RTK基地局 → TCP:2101 → 屋内サーバー → MAVLink → ドローン
```

### 4.3 GPS誤差比較実験 結果

#### サンプル1：DGPS（通常GPS・補正なし）

| 項目 | u-blox（基準） | Pixhawk（ドローン） |
|---|---|---|
| 測位方式 | DGPS | DGPS |
| 衛星数 | 12 | 24〜27 |
| 精度 | 〜1m | 〜1m |

| 統計量 | 値 |
|---|---|
| **平均水平誤差** | **1.13m** |
| 標準偏差 | 0.05m |
| CEP50 | 1.12m |

→ DGPS単独測位として**良好な精度**（公称精度0.5〜2mの範囲内）

#### サンプル2：DGPS + RTCM補正注入

| 項目 | 結果 |
|---|---|
| RTCM注入 | ✅ 正常動作（GPS_RTCM_DATA送信確認） |
| 水平推定誤差(eph) | 0.68m（サンプル1比 **40%改善**） |
| 測定安定性(σ) | 0.01m（サンプル1比 **80%改善**） |

> ⚠️ バッテリー再接続後にPixhawkのGPSが異常値を示したため、絶対位置の比較は参考値。相対的な精度改善効果は確認できた。

**詳細データ:** `docs/gps_comparison_sample1.md`
