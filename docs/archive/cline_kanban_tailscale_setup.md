# 作業指示書：Cline Kanban を Tailscale 経由で共有する

## 背景

これまで自作のKanbanアプリ（Flask + Vanilla JS、`~/kanban` 配下）を開発していたが、目的を整理した結果、
**「独自Kanbanアプリを作ること」自体が目的ではなく、「複数端末から同じタスク盤面・同じ履歴を見たい」ことが目的**だと判明した。

この目的は、自作アプリではなく **本家 Cline Kanban（`cline/kanban`, npmパッケージ `cline`）を1台のマシンで起動し、
Tailscale経由で他端末からアクセスする構成**で達成できる。

**自作Flaskアプリ（`~/kanban`）の開発はここで一旦停止する。** 今回のタスクはその代わりに、
Cline Kanban本体のセットアップと動作確認を行うことである。

## ゴール

- メイン機（サーバー役）で Cline Kanban を起動し、Tailscale IP にバインドする
- リモート端末（別のPC・スマホ等）から、同じTailscaleネットワーク経由でその盤面にアクセスできる
- 履歴・タスク状態が全端末で共有される（各端末で個別にKanbanを起動しない）
- リモートアクセスにpasscode認証がかかっている状態にする

## 前提環境

- Node.js 22以上がインストール済みであること
- サーバー役・リモート役の両端末に Tailscale がインストールされ、同じTailscaleアカウントでサインイン済みであること
  - 参考：以前の設定ファイルに Tailscale IP `100.75.83.95` が使われていた形跡があるため、既にTailscale自体は導入済みの可能性が高い

## 参照すべき公式ドキュメント

作業を始める前に、必ず以下の公式ページを確認すること：

**https://docs.cline.bot/kanban/remote-access**

このドキュメントには、Tailscale経由のリモートアクセス手順に加えて、Docker・ngrok・Cloudflare Tunnel・SSHトンネルなど他の共有方法や、
セキュリティ上の注意点（`--host 0.0.0.0` は同一LAN上の全端末に公開されるため信頼できるネットワークでのみ使うこと等）が記載されている。
以下の手順書はこのドキュメントの内容を踏まえて作成しているが、**バージョンアップ等でフラグ名や挙動が変わっている可能性があるため、
実際にコマンドを叩く前に該当ページの最新内容と照らし合わせること。**

## 作業手順

### 1. Cline CLI のインストール（サーバー役の端末）

```bash
npm i -g cline
cline --version
```

### 2. Tailscale IP の確認（サーバー役の端末）

```bash
tailscale ip -4
```

出力されたIPアドレスを記録する（以下 `<TAILSCALE_IP>` とする）。

### 3. Kanban対象にしたいgitリポジトリのルートに移動

```bash
cd <対象プロジェクトのパス>
```

### 4. Kanban を Tailscale IP にバインドして起動

```bash
kanban --host <TAILSCALE_IP>
```

- ポートを固定したい場合は `--port 3484` を明示的に追加してもよい（デフォルトは3484）
- 起動時のターミナル出力に **passcode** が表示されるはずなので、それを控えておく
- passcodeが表示されない・認証が無効になっている場合は、意図せず無認証で外部公開される状態なので、
  `kanban --help` でpasscode関連のフラグ（バージョンによって名称が異なる可能性あり）を確認し、認証が有効になっているか必ず確認すること

### 5. リモート端末からの接続確認

リモート端末のブラウザで以下にアクセス：

```
http://<TAILSCALE_IP>:3484
```

- passcodeの入力を求められたら、手順4で控えたものを入力
- サーバー役の端末で見えているタスク・履歴と同じものが表示されることを確認する

### 6. 動作確認項目

- [ ] リモート端末からタスクの作成ができる
- [ ] リモート端末で作成したタスクが、サーバー役の端末の画面にもリアルタイムに反映される
- [ ] サーバー役の端末を再起動しても、タスク履歴が消えずに残っている
- [ ] OSのファイアウォール（Windows Defender ファイアウォール／macOSファイアウォール）が3484番ポートの着信をブロックしていないか確認する

## トラブルシューティング

| 症状 | 確認すること |
|---|---|
| リモートからアクセスできない | サーバー役端末のファイアウォール設定、Tailscaleが両端末で同じアカウントに繋がっているか（`tailscale status`で確認） |
| passcodeなしでアクセスできてしまう | 使用しているKanbanのバージョンを確認し、認証を無効化するフラグを誤って付けていないか確認する |
| 履歴が相変わらず端末ごとに変わる | 各端末で個別に `kanban` を起動していないか確認する。起動すべきなのはサーバー役の1台のみ |
| ポート3484が使用中というエラー | 別プロセスが既にそのポートを使っている可能性がある。`--port auto` で空きポートを自動選択させるか、`lsof -i :3484`（Mac/Linux）で使用中のプロセスを確認する |
